
1) Введение
	Целью данной спецификации является разработка игрового приложения "Сапёр 3D" с использованием трехмерной графики.
	Продукт будет включать в себя начальное меню, способ выбора сложности, интерактивное игровое поле а также сообщения о статусе игры.
	В продукте не предусматривается наличие редактора уровней и настроек управления или графики.

2) Требования пользователя
2.1) Программные интерфейсы
	Для поддержки кроссплатформенности, взаимодействия с пользователем и расширения возможностей приложения была выбрана библиотека C++ Magnum. Она предоставляет высокоуровневые функции-обёртки средств OpenGL, добавляет средства организации сложных сцен и взаимодействий объектов, а также содержит простые интерфейсы к популярным устройствам пользовательского ввода.
2.2) Интерфейс пользователя
	UI будет представлен несколькими вариантами меню и одним игровым полем. Поле будет представлено трехмерным массивом полупрозрачных кубов-клеток. Клетки могут быть неоткрытыми, пустыми, содержать мину или указывать на мину рядом через свой цвет и цифру на гранях. Одновременно будет активен только один из горизонтальных слоев поля, пользователь может переходить на соседние слои. Неактивные слои рядом с активным будут менее заметны для лучшего обзора, а слои дальше от него не будут отображаться. Пользователь может взаимодействовать лишь с активным слоем через мышь для смещения камеры, вращения камеры и действий над клетками.
2.3) Харакеритики пользователей
	Целевой аудиторией данного приложения являются пользователи, имеющие достаточный уровень развития для понятия принципов действия стандартной версии игры "Сапёр".
2.4) Предложения и зависимости
	Требования к представлению игрового поля могут измениться в результате тестирования вариантов его внешнего вида с целью предоставления более ясной информации о состоянии поля.
	
3) Системные требования
3.1) Функциональные требования
	1. Главное меню должно содержать кнопки "Начать игру", "Выйти", а также выбор сложности - размеров поля и числа мин.
	2. Число мин должно быть ограничено с целью уменьшения вероятности победы на первом ходу или отсутствия хороших ходов.
	3. Игровое поле должно отображать количество оставшихся мин и прошедшее время в формате ССС.
	4. Фокус камеры должен быть ограничен границами поля.
	5. Камера должна свободно вращаться вокруг вертикальной оси, но иметь ограниченный подъем.
	6. Должен быть способ изменения масштаба камеры.
	7. После исхода игры пользователю должно показываться сообщение с результатами игры и выбор перезапуска или возвращения в главное меню.
3.2) Нефункциональные требования
3.2.1) Переносимость
	Библиотека Magnum является кроссплатформенной, но для работы с ней использууется язык C++. Поэтому для работы на другой системе достаточно перекомпиляции исходного кода.
